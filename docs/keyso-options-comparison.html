Keys.so: MCP vs Skills Comparison

Reference for choosing the smallest effective option per task.

Option Type Best for API breadth Typical calls Expected context load Tradeoff
MCP only
150 tools		 Raw tool surface Unknown or highly custom workflows Full API Any, no routing guardrails High
~6k-20k+ tokens		 Maximum flexibility, highest context cost.
Skill 1
keyso-api-router		 Wide router Multi-step analytics, reports, monitoring, compare, async flows Broad but guided 1-3 starter calls, then expand by family Medium
~2k-8k tokens		 Good balance of power vs context.
Skill 2
keyso-quick-audit		 Narrow workflow Fast domain/keyword snapshot + one follow-up Narrow (3-5 endpoints) 1 primary + max 1 extra Low
~0.8k-2.5k tokens		 Very fast, less suitable for deep analysis.
Skill 3
keyso-domain-dashboard-lite		 Micro skill Single domain snapshot only Ultra narrow (1 endpoint) Exactly 1 endpoint Very low
~0.4k-1.2k tokens		 Cheapest context, strict scope.

Choose MCP only

When task is unusual and no skill fit is clear.

Choose Skill 1

When user asks complete investigations across multiple report families.

Choose Skill 2

When user needs a quick answer around one domain or one keyword.

Choose Skill 3

When you only need baseline domain health in one call.
Context-load numbers are practical estimates for prompt+skill+routing overhead in typical sessions and may vary by client.
